the math

Every backward pass, by hand

No autograd anywhere. The gradient of attention, RMSNorm, SwiGLU, softmax, cross-entropy and the embedding scatter are each derived on paper and written out as code.

The softmax backward never materializes the Jacobian — it uses the s ⊙ (g − Σ s·g) identity so it stays batched. Cross-entropy supports ignore_index, which is what makes SFT possible: loss is computed on assistant turns only.

the gpu

A CUDA engine, mirrored 1:1

A standalone C++17/CUDA trainer at full feature parity with the NumPy one — same architecture, same init scales, same numerics, same checkpoints.

Twenty-three hand-written kernels — k_rmsnorm_backward, k_swiglu_backward, k_softmax_forward, k_embedding_backward and friends — with shared-memory block reductions and atomic gradient accumulation. cuBLAS handles the matmuls; everything around them is hand-rolled, including RAII device buffers that stop reallocating once training reaches steady state.

the tokenizer

Byte-level BPE, in C++

Not HuggingFace tokenizers — a from-scratch trainer and encoder. Merge selection runs off a max-heap with an inverted pair→word index and lazy deletion, so each merge only re-counts the words it actually touched.

256 byte tokens as the base vocabulary means zero OOV. Pretokenization is a hand-written UTF-8 state machine reproducing the GPT-4 splitting rules, right down to capping digit runs at three characters.

Training

Loss and perplexity, straight from the metrics.csv the trainer writes each step. Optimizer is plain SGD — hand-written, like everything else.
